# Doppel v0.1.0 Retrospective

Doppel v0.1.0 解决的问题很具体：传统测试能证明功能存在，却很难证明陌生用户是否能理解产品、找到入口并自然走完关键路径。Doppel 用 synthetic user 进入真实页面，把观察、点击、滚动、等待、停止原因和截图转成可复查的 artifacts，再用 judge criteria 生成体验评审报告。

## 当前实现范围

v0.1.0 的实现范围聚焦 Web 页面体验测试。它支持产品配置、persona 配置、单文件 judge skill、browser-use runtime、OpenAI 兼容模型、runtime fallback、多 persona batch、facts 提取、criteria 评估、Markdown/JSON 报告和点击目标标注。

当前版本已经具备完整 CLI 闭环：`validate` 负责配置校验，`run` 负责单 persona 运行，`batch` 负责多 persona 批量运行和汇总报告。

## ai-bot-cn 示例结果

`examples/ai-bot-cn` 被固定为 v0.1.0 官方展示样例。单 persona 示例保存了核心 artifacts：`session.json`、`facts.json`、`evaluation.json`、`report.md`、`report.json` 和关键截图。batch 示例保存了 `batch_report.md`、`batch_summary.json`，并为新手、设计师、产品经理和谨慎用户生成独立运行目录和独立报告。

这轮结果显示，`ai-bot.cn` 能被多个 persona 理解为 AI 工具导航与内容站点，搜索框、AI 设计工具、关于我们等入口能被不同角色识别和点击。报告同时指出首屏定位表达不够直接、部分路径缺少滚动发现、信任信号暴露不稳定、谨慎用户路径存在等待冗余。

## 已知限制

模型结构化输出仍不稳定，运行中会出现缺逗号、控制字符、超时等问题。browser-use 的元素编号可能因页面动态变化失效。事实提取仍依赖启发式文本规则，点击目标、产品理解和信任信号的抽取有误差。当前只验证了 Web 页面，尚未覆盖桌面应用、虚拟机、登录后流程和 CI/CD。

这些限制是 v0.1.0 的边界。它们不会推翻 MVP 闭环，但会影响日常重复运行的稳定性。

## 后续方向

后续工作应聚焦稳定化和可复用性：把 runtime guardrail 下沉到代码，增强首屏快照和页面文本证据，改进点击目标和滚动发现质量，支持 judge skill package，扩展 sample matrix，接入 CI/CD，提供跨版本 report comparison。

v0.1.0 的价值在于证明方向成立。下一阶段的价值在于让它从“能演示”进入“可稳定复用”。
